# Pythonic 教程 - 第二课：注释规范

## 一、注释基础语法
注释是代码中不执行的说明文字，核心作用是提升代码可读性，Python 分两种注释形式：

### 1. 单行注释
- 语法：以 `#` 开头，注释内容跟在后面
- 适用场景：单行说明、临时注释代码
- 示例：
```python
# 定义用户年龄（单行注释）
user_age = 18

# print(user_age)  # 注释掉临时不需要执行的代码
```

### 2. 多行注释
- 语法：用三引号（`'''` 或 `"""`）包裹多行内容
- 适用场景：大段说明、函数/类的开头注释
- 示例：

```python
"""
这是多行注释
可以写多行说明文字
用于解释一段代码的核心逻辑
"""
total_score = 95 + 88  # 计算总分
```

<div class="tip">
<strong>小提示：</strong>三引号本质是字符串，仅当它不赋值给变量、不作为函数参数时，才充当注释作用。
</div>

## 二、什么是“好的注释”
好注释不是复述代码，而是“解释代码的意图”，核心原则：
1. 不冗余：避免“注释说废话”（如 `# 给a赋值1`）
2. 讲原因：解释“为什么这么写”，而非“写了什么”
3. 语义化：清晰易懂，符合Pythonic风格

### 正反示例对比

<p><strong>不好的注释（冗余/无意义）：</strong></p>

```
a = 10  # 给a赋值10
# 循环10次
for i in range(10):
    print(i)
```

<p><strong>好的注释（解释意图）：</strong></p>

```
timeout = 10  # 接口请求超时时间（秒），根据业务要求设置
# 遍历打印前10个序号，用于生成订单编号前缀
for i in range(10):
    print(i)
```


## 三、函数的标准注释格式
Python 推荐的函数注释模板（清晰、易读，符合社区规范）：

```python
"""
函数功能的核心说明  

:param 参数名: 参数的含义/类型/取值范围
:return: 返回值的含义/类型（多返回值需分别说明）
"""
```

### 1. 单返回值函数
```python
def calculate_sum(num1, num2):
    """
    计算两个数字的和  

    :param num1: 第一个加数
    :param num2: 第二个加数
    :return: 两个数的求和结果
    """
    return num1 + num2
```

### 2. 多返回值函数
多返回值需明确说明每个返回值的含义，用逗号分隔并标注顺序：

```python
def get_user_info():
    """
    获取用户的基础信息  

    :param: 无参数
    :return: 用户名, 用户年龄, 手机号
    """
    name = "张三"
    age = 25
    phone = "13800138000"
    return name, age, phone
```

## 四、小结
- 注释语法：单行 `#`，多行 `'''/"""`
- 好注释：讲意图、不冗余、语义化
- 函数注释：按「功能说明+param+return」格式写，多返回值需逐个说明
