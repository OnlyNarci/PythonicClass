# Pythonic 教程 - 第一课：什么是Pythonic；变量命名规范

<strong>从萌新到python社区成员的必经之路</strong>


## 一、什么是 Pythonic？
### 核心定义
Pythonic 指**符合 Python 设计哲学**的代码风格与编程思路，核心是「优雅、简洁、可读」，遵循「用一种最好的方式解决问题」的原则。

### Pythonic 的优点
- 代码易读、易维护，符合社区共识
- 减少冗余，用最少的代码实现功能
- 贴合 Python 语言特性，运行更自然

### 快速查看 Python 设计哲学
在 Python 交互环境中执行以下代码，查看「Python 之禅」：
```python
import this
```

## 二、变量命名规范
### 1. 核心命名法
#### （1）下划线命名法（snake_case）
- 规则：小写字母 + 下划线分隔单词
- 适用场景：普通变量、函数、模块（Python 最常用）
- 示例：`user_name`、`age`、`total_score`

#### （2）大驼峰命名法（PascalCase）
- 规则：每个单词首字母大写，无下划线
- 适用场景：类名
- 示例：`UserInfo`、`StudentScore`

### 2. 实用小技巧
遇到不懂的英文单词，可通过「百度翻译」查询对应英文，保证命名语义化（如：用户年龄 → `user_age`，而非 `yonghunianling`）。

### 3. 对比小案例
<p><strong>不规范命名（语义模糊）：</strong></p>

```python
class yonghuxinxi:
    a = '小明'
    b = 14
    c = '篮球'
```

<p><strong>规范命名（语义清晰）：</strong></p>

```python
class UserInfo:
    name = 'Julina'
    age = 17
    hobbit = 'basketball'
````


## 三、小结
- Pythonic：追求简洁、优雅、符合Python设计哲学
- 变量命名：普通变量用 snake_case，类用 PascalCase，优先语义化
- 记住：好的命名让代码「自注释」，是 Pythonic 的第一步
